BeeBooking

Problem

The company recruits seasonal workers from Finland and other countries, but the application process was inefficient and fragmented. Employees applied via email without a standardized application form or processing system. Each application required an individual response, and employee data had to be manually transferred to the accounting system. When the employer organized transportation, passenger lists were also created manually, adding to the administrative burden.

Solution

We created a web application for submitting job applications, processing them, and managing recruitment workflows. The system includes an integrated ticket booking feature for employee transportation. Applicants can attach scans of required documents directly to their application forms. They automatically receive a notification about the change in the status of the application. Employee data can be exported to the accounting system automatically.

Result

The application streamlined the entire recruitment process, handling over 500 job applications annually. Manual data entry and email-based communication were replaced with an efficient, centralized system that saves time and reduces errors.

Styres

Problem

Small businesses needing warehouse management systems face a challenging market situation. Enterprise-level ERP solutions with WMS modules cost 100+ euros monthly and offer far more complexity than needed. Simpler alternatives priced at 18-24 euros per month have significantly limited features or hidden event-based charges. Many small businesses (providing storage services like tire warehouses, service stations, or frozen berry storage facilities) still rely on outdated methods such as spreadsheets due to the lack of affordable, targeted solutions.

Solution

We developed Styres, a modern SaaS warehouse management application designed specifically for small businesses. The platform provides essential modules including warehouse management, goods tracking, inventory control, customer management, and data import/export capabilities. The system features a simple, intuitive interface that delivers targeted information without unnecessary complexity. Optional modules for industry-specific features (such as tire storage services) allow for customization based on business needs.

Result

Styres delivers a targeted solution for small businesses seeking professional warehouse management without the burden of expensive, overcomplicated systems.

Technical Side

The architecture of the application supports horizontal scaling through multiple container instances and database schema separation for multi-tenant deployment. Cloud-hosted infrastructure. The web application supports QR and barcode scanning via the device camera and includes built-in label printer integration.

Time and Place

Developed 2023-2024 in Savonlinna, Finland. Primary customer: Autohuolto Kovanen OY.

SOP-Manager

Problem

The company's warehouse lacked an efficient system for locating components and tracking inventory. Workers spent unnecessary time searching for items, and real-time inventory monitoring was difficult. Although the company had an ERP system, it didn't provide dedicated tools for warehouse floor operations, slowing down production processes.

Solution

We developed SOP-Manager (Storage Order Pickup Manager), a web application that integrates directly with the company's existing ERP system and database. The application helps workers locate components in the warehouse, manage item pickups, and monitor total inventory balances. It streamlines warehouse operations through user-created templates that organize and optimize material handling processes. Built as a standalone module, the application communicates directly with the organization's database without disrupting existing workflows.

Result

The application successfully enhanced warehouse efficiency by providing workers with quick access to location data and inventory information. Template-based workflows reduced the time needed for routine picking operations, and real-time inventory tracking improved production planning accuracy.

Technical Side

The system uses ODBC drivers (unixODBC, MySQL-connector) to establish database links with the client's MariaDB database. Oracle ORDS acts as a proxy server managing application connectivity. The architecture maintains security through firewall configuration while enabling direct communication between the application and the client's existing database infrastructure. Progressive Web App (PWA) technology allows installation on devices for quick access.

Time and Place

Developed January-February 2025 in Savonlinna, Finland. Customer: eLive Ecosystem OY.

Production Label Terminals

Problem

Production tracking at the enterprise required workers to manually record their output, leading to inconsistencies and time waste. Employees needed a quick way to label manufactured products with tracking codes that would later be scanned in the warehouse for performance monitoring and inventory control.

Solution

We developed autonomous label printing terminals positioned directly at production workstations. Each terminal consists of a single-board computer (SBC) connected to a touchscreen display, RFID card reader, and label printer. Workers simply scan their personal RFID card containing their employee ID, enter the quantity of labels needed through the touch interface, and receive printed stickers immediately. The labeled products are later scanned in the warehouse, automatically recording both employee performance metrics and product tracking data.

Result

The terminals streamlined the production labeling process, eliminating manual recording and reducing workflow interruptions. Workers gained instant access to label printing at their workstations, while management received accurate, real-time production data.

Technical Side

Single-board computer running a custom application with touchscreen interface. System integrated with RFID card readers for employee identification and thermal label printers for sticker output. The terminals operate within the network communicating with the database to log production data.

Time and Place

2025 Savonlinna, Finland. Customer: JSM OY.

Video surveillance system

Problem

The customer needed comprehensive video surveillance coverage across multiple locations, including both indoor facilities and open outdoor areas. Existing security measures were insufficient for monitoring large territories and preventing incidents.

Solution

Result

High-quality recording and the alert system significantly improved security management.

Technical Side

Multi-channel DVRs configured for continuous high-resolution video recording and storage. Custom network configuration optimized for video streaming bandwidth and reliability. Alert system integrated with DVR software for motion detection and event notifications.
